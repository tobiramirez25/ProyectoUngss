En source, van a encontrar dos package y el archivo Main.
En el Main se crea una instancia del Tad principal y se utilizan los métodos para registrar proyecto y empleados para tener datos iniciales.

entidades 
Es donde deben colocar todas las clases generadas por Uds.
gui
Están todas las pantallas necesarias para usar los requerimientos del tp
Pantalla inicial con botones para entrar en las distintas opciones.
Nuevo empleado para cargar datos de empleado
Gestión empleados que permite ver los retrasos de los empleados registrados.
Nuevo proyecto para cargar datos de proyecto
Lista Proyectos para seleccionar el proyecto sobre el cual se quiere trabajar, luego se pasa a gestión

No es necesario entender el código de las pantallas solo respetar la interfaz provista para el TAD principal.
Se pueden cambiar los mensajes que se muestran, cuando se captura una Excepción

Ningún dato esta validado, salvo lo necesario para que no de errores de interfaz grafica. Todas las validaciones las debe realizar cada clase, asegurando que se cumpla el IREP 

Tener en cuenta que hay cambios en la interfaz del TAD principal, esta comentado cuales fueron los cambios.
